# cornell-ai-note
本项目为非技术背景产品经理利用 AI 辅助编程（Vibe Coding）独立落地的 PoC 验证版

## Project Vision

用「康奈尔式双栏 + 可控 AI 整理」把课堂速记变成可复习、可核对、可导出的学习材料，而不是另一份不可信的“代写作文”。

## Core Features

- **双栏对照**：左栏为课堂速记主画布，右栏为个人感悟/提示区；行高联动，减少阅读错位。
- **AI 结构化整理**：一键「魔法整理」，按行生成更易读的 Markdown 结构（标题层级、术语/重点标记），并生成课后反思三问与由标题抽取的思维导图导航。
- **原文锚点**：每一行同时保留原始速记与 AI 精修结果，支持在精修与原文之间切换核对，降低「AI 改完找不回原句」的焦虑。

## Tech Stack

- **前端框架**：React 18 + TypeScript  
- **构建与开发**：Vite 5  
- **样式**：项目内自定义 CSS（非 Tailwind）  
- **AI**：DeepSeek Chat Completions API（浏览器端携带用户自有 API Key 调用）  
- **本地能力**：Tesseract.js（图片 OCR）、html2canvas + jsPDF（导出 PDF）、LocalStorage（草稿与 Key 持久化）

## Why AI?

- **认知负荷**：课上时间紧，用户更需要「先记下来、再被帮一把理清结构」，而不是当场完成一篇完美文章。AI 负责把碎片信息整理成可扫读的结构，把脑力留给理解与追问。  
- **AI 幻觉与信任**：不把 AI 当作唯一真相来源；通过「逐行对应 + 原文可回看」把风险从「不可见改写」变成「可见、可核对」。右栏思考区刻意不进入模型输入，避免 AI 污染用户的独立反思。

## Product Manager's Note

- **需求减法**：早期很容易堆「知识图谱、自动关联全课程、多模型切换、云端协作」等大功能；本阶段明确只做「单页、本地草稿、单模型通路、可核对输出」，把复杂度压在用户能理解的闭环上。  
- **砍掉或推迟的设想**：例如全站知识图谱、跨笔记语义检索、服务端账号体系等——在「没有稳定数据模型与评测闭环」前先不做，避免功能看起来很全、体验却不可依赖。  
- **取舍原则**：凡是不能写进验收标准、不能在失败时保护用户原文的能力，默认不进 MVP。
  注：首次使用需在应用内配置 API Key；请勿将含密钥的 .env 提交到仓库。
